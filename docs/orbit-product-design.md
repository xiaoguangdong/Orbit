# Orbit / 星筹 产品设计文档

## 1. 产品定位

Orbit 是一款面向 Android 用户的习惯养成与目标打卡 App。中文名「星筹」表达「把每天的小行动筹成一片星图」：用户不只是勾掉清单，而是在一个可视化的个人轨道系统里持续积累、复盘、调整。

产品核心不是任务管理，而是「让长期习惯变得清晰、可执行、可反馈」。待办、四象限、倒数日、专注计时等能力作为辅助模块存在，服务于习惯目标。

## 2. 目标用户

### 2.1 核心用户

- 想建立稳定日常的人：运动、阅读、早睡、写日记、学习、冥想、喝水等。
- 需要轻量自律工具的人：不希望复杂项目管理，只需要每天打开、完成、看到进展。
- 有长期目标但容易中断的人：希望通过连续天数、提醒、统计、里程碑增强坚持感。
- 从滴答清单、Todoist、Habitify、Loop Habit Tracker 等工具迁移的人：熟悉清单与打卡，但希望更聚焦习惯。

### 2.2 使用场景

- 早上打开 App 查看今天要做的习惯。
- 完成运动、学习、阅读后快速打卡。
- 周末查看连续天数、完成率和中断原因。
- 创建一个新习惯，并设置频率、时间段、提醒和目标日。
- 通过桌面组件直接打卡。

## 3. 产品原则

- 今日优先：用户打开 App 后第一眼看到今天要完成什么，以及已经完成多少。
- 轻打卡：打卡必须是一秒内可完成的动作，长记录放在详情里。
- 长期反馈：用连续天数、完成率、星筹值、轨道进度表达积累。
- 可恢复：断签不是失败，App 应帮助用户复盘和恢复轨道。
- 不做重型待办：支持一次性事项，但不把项目管理作为核心。
- 任务为习惯服务：四象限任务用于每天聚焦优先级，不替代习惯系统。

## 4. 品牌与概念系统

### 4.1 命名

- 英文名：Orbit
- 中文名：星筹
- Slogan：Small actions, steady orbits.
- 中文表达：把每一天，筹成自己的星轨。

### 4.2 产品隐喻

- Habit / 习惯：一条 Orbit，代表一个长期运转的行为。
- Check-in / 打卡：一次点亮，代表当天轨道节点完成。
- Streak / 连续：当前轨道连续运转天数。
- Milestone / 里程碑：目标日、纪念日、阶段性成就。
- Star Credit / 星筹值：完成习惯后获得的成长积分，用于等级、徽章和成就。

### 4.3 视觉方向

- 基础风格：清爽、轻量、低干扰，延续截图中大字号、卡片、底部导航、浮动新增按钮的移动端习惯。
- 主色建议：Orbit Blue `#2F6BFF`，用于主要按钮、选中态、进度。
- 辅色建议：Mint `#24C99A` 表示完成，Amber `#F6B938` 表示提醒/里程碑，Coral `#FF6B6B` 表示中断/风险。
- 背景：浅灰蓝 `#F5F7FC`，卡片白色，避免重装饰。
- 字体策略：Android 默认使用系统字体，标题加粗，正文保持高可读性。
- 图标风格：圆形彩色图标表示习惯类别；功能入口使用线性图标。

## 5. 信息架构

### 5.1 底部导航

1. Today / 今日
2. Orbit / 轨道
3. Insights / 洞察
4. Milestones / 里程碑
5. Me / 我的

### 5.2 页面职责

- Today：展示今日习惯、打卡状态、日期切换、快速新增。
- Orbit：管理所有习惯，按时间段、分组、状态查看。
- Insights：统计、热力图、连续天数、完成率、趋势和复盘。
- Milestones：倒数日、已过纪念日、习惯目标日。
- Me：个人信息、等级、星筹值、会员、设置、导入导出。

## 6. 核心页面设计

### 6.1 Today / 今日

目标：让用户快速知道今天要做什么，并完成打卡。

页面结构：

- 顶部：菜单按钮、标题 Today / 今日、灵感/建议入口、更多菜单。
- 日期条：横向展示最近 7 天，默认选中今天。
- 概览卡：今日完成度、剩余数量、连续保持中的习惯数量。
- 分组列表：按 Morning / Afternoon / Evening / Anytime 展示。
- 已完成分组：默认折叠，可展开查看已打卡项目。
- 浮动按钮：新增习惯或快速任务。
- 子视图切换：Habits / Quadrants。

习惯卡片信息：

- 习惯图标
- 习惯名称
- 当前连续天数
- 今日状态：未打卡、已打卡、跳过、失败
- 可选信息：提醒时间、目标量、备注标记

关键交互：

- 点击卡片左侧状态按钮：立即打卡。
- 点击卡片主体：进入习惯详情。
- 长按卡片：打开快捷操作，包括跳过、补卡、编辑、归档。
- 下拉刷新：同步数据。
- 日期切换：查看历史某日的计划和完成情况。
- 切换到 Quadrants：进入当天优先级任务视图，按四象限管理一次性事项。
- Today 更多菜单支持：Hide details、Hide completed、Sort incomplete first、Show/Hide date strip。

### 6.1.1 Quadrants / 四象限任务

目标：承接截图中的艾森豪威尔四象限，帮助用户把一次性事项和短期行动按优先级落位。

象限定义：

- I 重要且紧急：必须尽快处理。
- II 重要不紧急：长期价值最高，优先投入。
- III 不重要但紧急：能委托、能快速清掉的打扰项。
- IV 不重要不紧急：低优先级、可删除或延后。

页面结构：

- 顶部：标题 Today，切换到 Quadrants。
- 概览卡：Active Tasks、Completed Tasks、操作提示。
- 象限棋盘：2 x 2 卡片布局，每张卡片展示对应象限的任务列表。
- 浮动按钮：新增任务，默认可直接指定象限。

任务卡片信息：

- 任务标题
- 简短描述
- 截止日期
- 完成状态
- 编辑、删除入口

关键交互：

- 点勾选按钮：完成/恢复任务。
- 点任务主体：编辑任务。
- 新建任务时必须选择象限，可选描述和截止日期。
- 已完成任务默认遵循全局 `Hide completed` 设置。
- 任务截止日期支持 `Overdue / Today / Upcoming / Someday` 状态提示。

### 6.2 Orbit / 轨道

目标：集中管理全部习惯。

页面结构：

- 顶部：标题 Orbit / 轨道，筛选，排序，搜索。
- Tab：Active / 坚持中，Archived / 已归档。
- 分组：Morning、Afternoon、Evening、Anytime、自定义分组。
- 习惯列表：展示图标、名称、频率、当前连续、最好连续。

支持视图：

- Group View：按分组展示。
- Compact View：隐藏详情，提升列表密度。
- Calendar View：按月查看习惯完成分布。

### 6.3 Add Habit / 新建习惯

目标：用最少输入创建可执行习惯。

字段：

- 名称：必填，例如 Run, Read, Journal。
- 图标与颜色：必填，有默认值。
- 频率：每天、每周几次、指定星期、自定义间隔。
- 时间段：早上、下午、晚上、任意时间。
- 目标量：无目标、次数、分钟、数量、距离。
- 提醒：可选，支持多个提醒时间。
- 开始日期：默认今天。
- 结束条件：长期坚持、目标日期、完成次数。
- 备注：可选。

模板：

- Drink Water
- Read 20 Minutes
- Run 3 km
- Sleep Before 23:00
- Write Journal
- Call Family
- Deep Work

### 6.4 Habit Detail / 习惯详情

目标：展示一个习惯的状态、历史和设置。

内容：

- 顶部：图标、名称、当前状态、编辑入口。
- 核心指标：当前连续、最好连续、总完成、近 30 天完成率。
- 日历热力图：按天显示完成、跳过、失败、未计划。
- 记录列表：每次打卡时间、备注、数值。
- 操作：打卡、补卡、跳过、失败、归档、删除。
- 补卡记录需在历史列表中明确标注 Backfilled。

### 6.5 Insights / 洞察

目标：帮助用户理解坚持情况，而不是只展示数字。

模块：

- Weekly Review：本周完成率、最佳习惯、风险习惯。
- Streaks：当前连续榜、历史最佳连续榜。
- Heatmap：按月/年查看完成密度。
- Recovery：中断后恢复次数、最长空窗。
- Star Credit：星筹值、等级、徽章。

### 6.6 Milestones / 里程碑

目标：承接截图中的倒数纪念日能力，并与习惯目标关联。

类型：

- Countdown：距离某日期还有 N 天。
- Since：从某日期已过去 N 天。
- Habit Goal：习惯目标节点，例如坚持跑步 100 天。

卡片信息：

- 图标
- 名称
- 剩余/已过天数
- 关联习惯
- 到期提醒

### 6.7 Me / 我的

内容：

- 头像、昵称、等级、星筹值、徽章。
- 会员入口。
- 功能模块管理。
- 外观设置。
- 日期与时间设置。
- 声音、提醒与通知。
- 桌面小组件。
- 数据同步与导入导出。
- 新手指南、帮助与反馈。
- 登录、云同步、会员能力在 MVP 中仅保留接口占位，不提供真实后端流程。

## 7. 核心交互规则

### 7.1 打卡状态

- 未打卡：当天计划内但未完成。
- 已打卡：当天完成。
- 跳过：用户主动跳过，不计为失败，可配置是否中断连续。
- 失败：当天计划内未完成，计入中断。
- 补卡：允许补录历史日期，需记录补卡标记。

### 7.2 连续天数计算

- 只计算计划日。
- 非计划日不影响连续。
- 跳过默认不中断，但在统计中单独记录。
- 失败会中断连续。
- 补卡后需要重新计算相关连续天数。

### 7.3 星筹值

星筹值用于成长反馈，不应影响核心功能。

建议规则：

- 完成普通习惯：+10
- 完成带目标量习惯：+12
- 连续 7 天额外：+30
- 连续 30 天额外：+120
- 补卡完成：+3
- 跳过：+0
- 失败：+0

## 8. MVP 范围

必须包含：

- 习惯创建、编辑、归档、删除。
- 今日习惯列表与打卡。
- 最近 7 天日期切换。
- 分组展示。
- 连续天数与总完成次数。
- 四象限任务创建、编辑、完成、删除。
- 提醒通知。
- 习惯详情与基础日历。
- 本地数据持久化。
- 基础设置。

暂不包含：

- 社交分享。
- 四象限拖拽重排、批量移动、委托流。
- 番茄专注。
- 会员体系。
- 多端同步。
- 高级主题。
